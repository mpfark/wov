## Goal

Bring service nodes to life with named NPCs (shopkeepers, smiths) generated by AI. Talking to them opens the relevant service window directly. Detach the Soulforge from a single dedicated NPC and make it a tab inside any blacksmith flagged as "soulforge-capable".

## What you'll see in-game

- **Every vendor** has a named shopkeeper (e.g. *Hilda the Salt-Tongued*) with a one-line flavor description. Clicking **Talk** opens the Vendor panel directly, with the NPC's name shown as the panel subtitle.
- **Every blacksmith** has a named smith (e.g. *Old Brann Iron-Hand*). Clicking **Talk** opens the Blacksmith panel directly.
- A small set of blacksmiths flagged as **soulforge-capable** show a third tab inside the Blacksmith panel: **Repair / Forge / ⚒️ Soulforge** — gated by level (40 for Crown, 42 for Soulforged) with the existing "not yet worthy" message otherwise.
- *The Soulwright* NPC at The Deep-Core Forge stays as flavor (no longer required to forge).

## How it works

### 1. New `is_soulforge` node flag
- Migration adds `nodes.is_soulforge boolean DEFAULT false`.
- Admin Node Editor gets a checkbox alongside the other service flags.
- Backfill The Deep-Core Forge as the first soulforge-capable blacksmith.

### 2. AI service-NPC generation
- New edge function `ai-generate-service-npc` (steward-gated, rate-limited, follows the `ai-world-builder` pattern). Inputs: `node_id`. It fetches node name, area, region, region/area flavor, and service flags, then asks Lovable AI (`google/gemini-3-flash-preview`, structured tool-calling) to return `{ name, description, dialogue }` matching the node's tone and service role. Inserts into `npcs` and tags the dialogue with a service intent (see #3).
- New admin button **"Generate Service NPC"** in the Node Editor for any service node missing one.
- One-time **backfill script** (run via `code--exec` against the AI Gateway) walks every `is_vendor=true` or `is_blacksmith=true` node with no NPC and calls the same generator, with a 1s delay between requests to stay within rate limits.

### 3. NPC → service routing (no extra dialog step)
- Add a non-breaking `npcs.service_role text` column: `vendor`, `blacksmith`, or `null` (regular flavor NPC). Generated service NPCs get the matching role automatically; existing NPCs default to `null` and keep current dialog-only behavior.
- In `GamePage.tsx`, the `onTalkToNPC` handler checks `npc.service_role`:
  - `vendor` → `setVendorOpen(true)` + sets a `vendorNpc` state used as panel subtitle ("*Hilda the Salt-Tongued tilts a knowing grin...*").
  - `blacksmith` → `setBlacksmithOpen(true)` similarly.
  - `null` (or any other) → existing `NPCDialogPanel`.
- `ServicePanelShell` already accepts a `subtitle` slot, so no new layout is needed.

### 4. Soulforge as a Blacksmith tab
- `BlacksmithPanel` becomes 3-tab when the current node has `is_soulforge=true`: **Repair / Forge / ⚒️ Soulforge**. Otherwise the current 2-tab layout stays untouched.
- The Soulforge tab embeds the existing `SoulforgeDialog` body (mode picker → forge UI) inline into the shell's left/right/footer slots — same logic, no new server work needed (the `soulforge-item` edge function already enforces level + ownership server-side).
- `GamePage.tsx` no longer routes `talkingToNPC?.name === 'The Soulwright'` to `SoulforgeDialog`; The Soulwright reverts to a normal flavor NPC at his node.

### 5. Node icons
- The node header icon row (`NodeView.tsx`) gets a `⚒️` icon when `is_soulforge=true`, next to the existing 🔨 / 🪙 / 🌀 icons.

## Backfill (one-time, after merge)

Run the AI generator against every existing vendor/blacksmith node with no NPC. There are 16 such nodes today (see investigation), so cost is bounded. The script:
1. Lists service nodes with `npc_count = 0`.
2. Calls `ai-generate-service-npc` per node with 1s spacing.
3. Logs successes/failures so any can be re-run.

## Out of scope

- Inn / Teleport / Trainer NPCs — same pattern can extend later, but this pass focuses on Vendor + Blacksmith + Soulforge as requested.
- Custom illustrations for the new NPCs.
- Voice/dialogue trees beyond a single flavor line.

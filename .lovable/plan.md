

## Audit findings

I scanned all common/uncommon items in the database against the rename and the stat budget formula.

### Issue 1 — Potions/foods got swept into the rename (16 consumables)

The original `ai-item-rename` violator detector matched any name containing the word `the`, `of`, or a lyrical suffix. That accidentally caught consumables like "Potion of Healing", "Potion of the Crimson Dawn", and food items, which the AI then renamed into the equipment-style format:

| Looks like a renamed potion/food | Should be |
|---|---|
| `Hardened Iron Trinket` (hp_regen 3, L15) | a food/potion name |
| `Reinforced Steel Trinket` (hp 66, L35) | a potion name |
| `Sturdy Health Potion`, `Heavy Health Potion` | fine names but show the AI touched them |
| `Crude Potion`, `Potion`, `Apple`, `Pear`, `Biscuits`, `Sausage`, `Steak`, `Oranges`, `Simple Ration` | likely renamed away from richer originals |

The 4 surviving "Potion of …" names slipped through because of varied phrasing.

### Issue 2 — 44 equipment items underspend their stat budget

Audit of all 316 common+uncommon equipment items using the canonical formula `floor(1 + (level-1) * 0.3 * rarity_mult * hands_mult)`:

- **248 exact** (78%) ✅
- **44 underspent** (14%) — missing 1 to 6 points  
- **24 overspent** (8%) — over budget by 1+ points

Worst offenders: `Galestring Longbow` (L25 2H uncommon, spent 11/17 — 6 short), `Stormstring` (5 short), `Earthrend Greataxe` (4 short). Most are tier-30/35 weapons.

These were generated by the original Forge before the budget rules were tightened — the rename pass only changed names, not stats, so the gaps are still visible.

## Plan

### Part A — Fix the consumables (one-shot SQL, no AI)

Restore canonical names from a hand-curated map. The 16 affected consumables are a small, fixed set, so an AI pass adds risk for no benefit. I will:

1. Read the current name + level + stats of each consumable.
2. Apply a deterministic mapping based on level + stat type:
   - `hp_regen` foods by level: L1 Bread → L5 Apple → L10 Pear → L15 Cheese Wheel → L20 Sausage → L25 Steak → L30 Roast Chicken → L35 Honey Cake → L40 Royal Feast (etc.).
   - `hp` potions by level: L5 Minor Healing Potion → L10 Lesser Healing Potion → L15 Healing Potion → L20 Greater Healing Potion → L25 Major Healing Potion → L30 Superior Healing Potion → L35 Master Healing Potion → L40 Grand Healing Potion.
3. Run a single `UPDATE` migration so it's reviewable and reversible.

I'll show you the proposed mapping before applying.

### Part B — Fix the rename function so it never touches consumables again

In `supabase/functions/ai-item-rename/index.ts`, change the items query from `.in("rarity", ["common","uncommon"])` to also require `.eq("item_type", "equipment")`. One-line guard.

### Part C — New admin tool: "Rebalance Stats"

A new edge function `ai-item-rebalance` (mirrors the rename tool) that:

1. Finds equipment where `spent < budget` (under) or `spent > budget` (over).
2. Sends each to Gemini with the original stats, slot, level, rarity, hands, and the canonical budget. Asks the model to redistribute to **exactly** the budget while:
   - preserving the dominant stat focus (e.g. dex weapon stays dex-primary),
   - respecting per-stat caps (`4 + floor(level/4)` for primaries, etc.),
   - keeping ≥2 different stats on equipment (Forge rule).
3. Validates the returned stats against budget and caps before writing. Skips on failure.
4. Supports `dry_run: true` to preview before applying.

Add a **"Rebalance Stats"** button next to "Rename Legacy" in `ItemManager.tsx` toolbar with the same dry-run-then-confirm flow you already saw with the rename tool.

### Out of scope
- Unique / soulforged items (their stats are intentional, hand-forged).
- Item descriptions, illustrations, slots, hands, value (untouched).
- The 24 overspent items are also fixed by Part C — same tool handles both directions.

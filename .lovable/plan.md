

# Remove PWA Service Worker, Keep Installability

## Problem

The service worker — even with `NetworkOnly` caching — gets cached by the browser itself. A stale worker can intercept requests and serve an outdated shell, which is why players see v0.8.1 instead of v0.9.3. Since the game is fully online-only, there's no benefit to having a service worker at all.

## Solution

Strip out the service worker registration while keeping the web app manifest so the game remains installable (Add to Home Screen).

## Changes

### 1. `vite.config.ts` — Remove VitePWA plugin
- Remove the `VitePWA` import and its entire config block from the plugins array.
- Remove `vite-plugin-pwa` from `package.json` dependencies.

### 2. `src/main.tsx` — Unregister any existing service workers
Add a cleanup block that unregisters all previously installed service workers so returning players aren't stuck on old cached versions:

```ts
navigator.serviceWorker?.getRegistrations().then(regs =>
  regs.forEach(r => r.unregister())
);
```

### 3. `src/components/PwaUpdatePrompt.tsx` — Delete file
No longer needed since there's no service worker to detect updates from.

### 4. `src/App.tsx` — Remove PwaUpdatePrompt
Remove the import and `<PwaUpdatePrompt />` from the component tree.

### 5. `src/vite-env.d.ts` — Remove PWA type reference
Remove `/// <reference types="vite-plugin-pwa/react" />`.

### 6. `index.html` — Keep manifest link
The existing `<link rel="manifest" href="/manifest.webmanifest" />` stays for installability. Create a static `public/manifest.webmanifest` with the same manifest content currently generated by VitePWA (name, icons, theme_color, display: standalone).

## What This Fixes

- No more stale service workers serving old app versions
- Every page load always fetches the latest code from the server
- Game remains installable via Add to Home Screen
- Existing users' stale workers get cleaned up on next visit
